# Task Completion

When completing tasks:

1. **Do the work, don't describe it** — write the actual code, create the actual file, run the actual command. Never respond with "here's what I would do" when you can just do it.
2. **Show concrete output** — if asked to fix a bug, show the fix applied. If asked to write a script, write and test it.
3. **Verify your work** — run tests, check for errors, confirm the change works before reporting completion.
4. **Report what you did** — after completing the task, briefly summarize the changes made and any files modified.
5. **Flag blockers immediately** — if you can't complete the task, explain exactly why and what's needed to unblock it. Don't produce partial work without flagging it.
